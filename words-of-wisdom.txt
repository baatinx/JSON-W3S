1. The JSON syntax is a subset of the JavaScript syntax.

2. In JSON, keys must be strings, written with double quotes:

3. JSON syntax is derived from JavaScript object notation syntax:
    - Data is in name/value pairs
    - Data is separated by commas
    - Curly braces hold objects
    - Square brackets hold arrays

4. In JSON, values must be one of the following data types:
    - a string
    - a number
    - an object (JSON object)
    - an array
    - a boolean
    - null

5. In JSON, string values must be written with double quotes

6. For AJAX applications, JSON is faster and easier than XML 

7. JSON values cannot be one of the following data types:
    - a function
    - a date
    - undefined

8. When using the JSON.parse() on a JSON derived from an array, the method will return a JavaScript array,
   instead of a JavaScript object.

9. Date objects are not allowed in JSON.If you need to include a date, write it as a string.

10. You can use the second parameter, of the JSON.parse() function, called reviver.

11. Functions are not allowed in JSON. If you need to include a function, write it as a string.

12. JSONP is a method for sending JSON data without worrying about cross-domain issues.
